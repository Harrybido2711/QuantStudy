## Definition

**Latency** is the time elapsed from the start of a request to its completion. In distributed systems and high-frequency trading, latency is typically measured in:

- **Milliseconds (ms)**: Web APIs, databases
- **Microseconds (μs)**: Low-latency trading systems
- **Nanoseconds (ns)**: Co-located HFT, custom FPGA systems

## Percentile Latencies

Never summarize latency with mean alone. Use **percentiles**:

| Metric | Meaning |
|--------|---------|
| p50 (median) | Typical case |
| p95 | 95% of requests are faster than this |
| p99 | 99% are faster — the "tail" |
| p99.9 | The extreme tail — often 10–100× p50 |

> "The 99th percentile determines the user experience of 1% of requests — which at 1 million RPS is 10,000 users per second."

## Sources of Latency

```
Network:     ~1ms per 100km (speed of light)
SSD:         ~100μs read
DRAM:        ~100ns read
L3 cache:    ~10ns
L1 cache:    ~1ns
```

## Latency in HFT

In high-frequency trading, latency is measured **round-trip** (RTT):

- **Exchange co-location**: 1–10μs
- **Cross-datacenter**: 50–500μs
- **Retail broker API**: 10–100ms

Co-located strategies have structural advantages that cannot be overcome by better signals alone at sub-millisecond horizons.

## Optimization Strategies

1. **Reduce network hops**: Co-location, edge computing
2. **CPU affinity**: Pin threads to cores, avoid context switching
3. **Kernel bypass**: DPDK, RDMA for zero-copy networking
4. **Lock-free data structures**: Avoid mutex contention
5. **Cache alignment**: Prevent false sharing
